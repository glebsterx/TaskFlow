# TeamFlow — Overview

TeamFlow — локальный инструмент фиксации договорённостей команды через Telegram с веб-просмотром состояния задач.

Проект предназначен для маленьких команд (3–5 человек), работающих без формальных процессов.

## Основные цели

- фиксировать задачи, озвученные в чате
- хранить недельные договорённости
- видеть текущий статус работы
- не внедрять таск-трекер
- работать локально без внешних сервисов

## Основные интерфейсы

1. Telegram Bot — основной интерфейс работы
2. Web UI — read-only обзор состояния

## Архитектурный стиль

Modular Monolith с разделением на слои:

- Transport
- Application
- Domain
- Persistence

## Хранилище

SQLite (единая база для всех интерфейсов)
